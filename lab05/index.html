Lab 02 Reflection

My first webpage! At first, the process seemed daunting, with so many tags and structures to remember, but breaking it down step by step made it manageable. Learning how to properly structure an HTML document helped me understand how webpages are built and displayed.

Lab 03 Reflection

This lab introduced me to CSS, and I love how powerful it is for controlling the look of a webpage. Connecting an external stylesheet to my HTML file felt easy to maintain. I could change colors, fonts, and spacing all in one place without touching the HTML structure. I found the difference between ID and class selectors. I was confused about when to use padding versus margin but I searched online and read about it. Resources that helped me the most were MDN Web Docs and the YouTuber Dave Gray.

New Things I Learned

  1. How to link an external CSS file using the <link> element in <head>.
  2. How to style elements using selectors like body, h1, p, and a.
  3. The difference between ID selectors (#) and class selectors (.).
  4. The difference between padding and margin.

Lab 05 Reflection

In this lab I learned about layout and responsiveness. I practiced using float, flexbox, and grid to arrange elements on a page. Flexbox made it easy to line items up in a row or column, while grid let me control both rows and columns at the same time. I also learned about media queries, which let me change the layout depending on the screen size. This is how websites look good on both phones and computers.

Best Design Practices I Learned

  1. Always include the viewport meta tag so the page scales correctly on mobile devices.
  2. Design mobile first, then use media queries to expand the layout for larger screens.
  3. Use flexbox for one-directional layouts like navigation menus and card rows.
  4. Use grid when you need both rows and columns at the same time.
  5. Keep font sizes readable on small screens and increase them for larger screens.
